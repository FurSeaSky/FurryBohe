## 二、目录结构

### 2.1 语言文件存放位置

```
src/main/resources/
└── assets/
    └── 你的modid/
        └── lang/                    ← 语言文件存放目录
            ├── en_us.json           ← 英文（源语言，基准）
            ├── zh_cn.json           ← 简体中文
            ├── zh_tw.json           ← 繁体中文（如有）
            ├── ja_jp.json           ← 日语（如有）
            └── ...                  ← 其他语言
```

### 2.2 语言代码对照表

| 语言 | 代码 | 文件名 |
|------|------|--------|
| 英文（美国）| `en_us` | `en_us.json` |
| 简体中文 | `zh_cn` | `zh_cn.json` |
| 繁体中文 | `zh_tw` | `zh_tw.json` |
| 日语 | `ja_jp` | `ja_jp.json` |
| 韩语 | `ko_kr` | `ko_kr.json` |
| 法语 | `fr_fr` | `fr_fr.json` |
| 德语 | `de_de` | `de_de.json` |

| 西班牙语 | `es_es` | `es_es.json` |
| 葡萄牙语 | `pt_br` | `pt_br.json` |


## 三、语言文件格式

### 3.1 JSON 格式规范

```json5
{
  "key": "翻译文本",
  "key2": "翻译文本2"
}
```

> ⚠️ **注意**：
> - 使用双引号 `"`，不用单引号 `'`
> - 每行末尾有逗号（最后一行不加逗号）
> - 编码：**UTF-8**

### 3.2 完整示例

```json5
{
  "item.modid.steel_ingot": "Steel Ingot",
  "item.modid.steel_sword": "Steel Sword",
  "block.modid.steel_block": "Block of Steel",
  "block.modid.iron_ore": "Iron Ore",
  "itemGroup.modid.main": "My Mod",
  "key.modid.open_menu": "Open Menu",
  "key.modid.open_menu.description": "Opens the main menu",
  "subtitles.modid.explosion": "Explosion",
  "death.attack.modid.laser": "%s was vaporized by a laser"
}
```

### 3.3 键名（Key）命名规则

| 类型 | 格式 | 示例 |
|------|------|------|
| 物品 | `item.modid.名称` | `item.modid.steel_ingot` |
| 方块 | `block.modid.名称` | `block.modid.steel_block` |
| 物品组 | `itemGroup.modid.名称` | `itemGroup.modid.main` |
| 按键绑定 | `key.modid.功能` | `key.modid.open_menu` |
| 按键描述 | `key.modid.功能.description` | `key.modid.open_menu.description` |
| 字幕 | `subtitles.modid.事件` | `subtitles.modid.explosion` |
| 死亡信息 | `death.attack.modid.原因` | `death.attack.modid.laser` |
| GUI | `gui.modid.组件` | `gui.modid.title` |
| 提示信息 | `message.modid.内容` | `message.modid.welcome` |
| 状态效果 | `effect.modid.名称` | `effect.modid.bleeding` |

> ⚠️ 所有 Key 必须 **全小写 + 下划线**，不要用大写字母


## 四、翻译规范

### 4.1 翻译原则

| 原则 | 说明 |
|------|------|
| **准确** | 忠实表达原文含义，不随意增减内容 |
| **简洁** | 游戏内 UI 空间有限，尽量简短 |
| **统一** | 同一术语在全模组保持翻译一致 |
| **原版风格** | 符合 Minecraft 原版翻译风格 |
| **文化适配** | 符合中文语言习惯，不直译生硬 |

### 4.2 术语对照表

#### 通用术语

| 英文 | 简体中文 | 备注 |
|------|----------|------|
| Block | 方块 | 保持与 Minecraft 一致 |
| Item | 物品 | 保持与 Minecraft 一致 |
| Entity | 实体 | 保持与 Minecraft 一致 |
| Ingot | 锭 | 标准译法 |
| Ore | 矿石 | 标准译法 |
| Sword | 剑 | 标准译法 |
| Pickaxe | 镐 | 标准译法 |
| Axe | 斧 | 标准译法 |
| Shovel | 锹 | 标准译法 |
| Helmet | 头盔 | 标准译法 |
| Chestplate | 胸甲 | 标准译法 |
| Leggings | 护腿 | 标准译法 |
| Boots | 靴子 | 标准译法 |
| Crafting | 合成 | 保持与 Minecraft 一致 |
| Smelting | 烧炼 | 保持与 Minecraft 一致 |
| GUI | 界面 | 或“图形用户界面” |
| Tooltip | 提示文本 | 鼠标悬停显示的说明 |
| Durability | 耐久度 | 标准译法 |
| Enchantment | 附魔 | 保持与 Minecraft 一致 |
| Potion | 药水 | 标准译法 |
| Particle | 粒子 | 标准译法 |
| Dimension | 维度 | 保持与 Minecraft 一致 |
| Biome | 生物群系 | 保持与 Minecraft 一致 |
| Structure | 结构 | 标准译法 |

#### 模组特定术语（根据实际内容填写）

| 英文 | 简体中文 | 备注 |
|------|----------|------|

| （模组方块名） | （中文译名） | 待补充 |

> 💡 建议创建一个术语表 Excel 文件放在项目根目录，方便团队统一翻译。

### 4.3 翻译风格示例

```json5
// en_us.json
{
  "item.modid.steel_sword": "Steel Sword",
  "item.modid.steel_sword.tooltip": "A durable sword made of steel",
  "message.modid.welcome": "Welcome to the world!"
}
```

```json5
// zh_cn.json
{
  "item.modid.steel_sword": "钢剑",
  "item.modid.steel_sword.tooltip": "由钢制成的耐用剑",
  "message.modid.welcome": "欢迎来到这个世界！"
}
```

### 4.4 占位符处理

Minecraft 的翻译文本中经常包含占位符，**必须保留原样**：

| 占位符 | 含义 | 示例 |
|--------|------|------|
| `%s` | 字符串占位符 | `"%s 被 %s 杀死了"` |
| `%d` | 数字占位符 | `"剩余 %d 秒"` |
| `%1$s`, `%2$s` | 指定位置的字符串占位符 | 用于多个变量的情况 |
| `%%` | 百分号本身 | `"有 50%% 的概率"` |

**示例：**

```json5
// en_us.json
{
  "death.attack.laser": "%s was vaporized by a laser"
}
```

```json5
// zh_cn.json（正确）
{
  "death.attack.laser": "%s 被激光蒸发了"
}
```

```json5
// zh_cn.json（错误 ❌ 丢失了 %s）
{
  "death.attack.laser": "被激光蒸发了"
}
```

### 4.5 数量不敏感的文本

对于英文中需要区分单复数的文本，中文不区分：

```json5
// en_us.json
{
  "item.modid.steel_ingot": "Steel Ingot"
  // 单复数写法不同：1 steel ingot / 2 steel ingots
}
```

```json5
// zh_cn.json
{
  "item.modid.steel_ingot": "钢锭"
  // 中文保持统一：1 个钢锭 / 2 个钢锭
}
```

### 4.6 颜色代码处理

如果 Key 值中包含颜色代码（`§`），**保留不变**：

```
示例：在 en_us.json 中
"gui.modid.title": "§lMy Mod §rSettings"

翻译时仅翻译文本部分，不改变颜色代码：
"gui.modid.title": "§l我的模组 §r设置"
```

| 颜色代码 | 效果 | 颜色代码 | 效果 |
|----------|------|----------|------|
| `§0` | 黑色 | `§8` | 深灰色 |
| `§1` | 深蓝 | `§9` | 蓝色 |
| `§2` | 深绿 | `§a` | 绿色 |
| `§3` | 青色 | `§b` | 亮青色 |
| `§4` | 深红 | `§c` | 红色 |
| `§5` | 深紫 | `§d` | 亮紫 |
| `§6` | 金色 | `§e` | 黄色 |
| `§7` | 灰色 | `§f` | 白色 |
| `§k` | 随机字符 | `§r` | 重置 |
| `§l` | 加粗 | `§o` | 斜体 |
| `§n` | 下划线 | `§m` | 删除线 |


## 五、提交流程

### 5.1 提交前检查清单

- [ ] 文件编码为 **UTF-8**（无 BOM）
- [ ] JSON 格式正确（逗号、引号、括号）
- [ ] 所有 Key 与 `en_us.json` 保持一致
- [ ] 占位符（`%s`、`%d` 等）完整保留
- [ ] 颜色代码（`§`）完整保留
- [ ] 术语与团队术语表一致
- [ ] 翻译简洁，符合游戏场景
- [ ] 文件放在正确的目录 `assets/modid/lang/` 下

### 5.2 提交信息格式

```bash
i18n(zh): 更新中文翻译

示例：
i18n(zh): 新增钢铁系列物品翻译
i18n(ja): 更新 UI 界面日语翻译
i18n(zh): 修复工具提示翻译错误
```

### 5.3 PR 标题格式

```bash
[i18n] 更新中文翻译
[i18n] 添加日语翻译文件
```

### 5.4 PR 描述模板

```markdown
## 变更类型
- [x] i18n（翻译）

## 变更说明
（描述更新了什么翻译）

## 变更文件
| 文件 | 变更说明 |
|------|----------|
| zh_cn.json | 新增 15 条翻译 |
| ja_jp.json | 新增文件，全部翻译完成 |

## 翻译对照表
| Key | 英文 | 中文 |
|-----|------|------|
| item.modid.steel_ingot | Steel Ingot | 钢锭 |
| block.modid.steel_block | Block of Steel | 钢块 |

## 审核清单
- [ ] JSON 格式正确
- [ ] 占位符保留完整
- [ ] 术语与术语表一致
```


## 六、添加新语言

### 6.1 新增语言文件

如需添加新的语言支持，按以下步骤：

**第 1 步：创建新文件**

```bash
# 在 lang 目录下创建对应的语言文件
touch assets/modid/lang/ja_jp.json   # 日语示例
```

**第 2 步：复制 `en_us.json` 内容**

```
复制所有 Key，将翻译值替换为目标语言
```

**第 3 步：提交**

```bash
i18n(ja): 添加日语翻译
```


## 七、常见问题

### 7.1 游戏中显示为 Key 原文（如 `item.modid.xxx`）

```
原因：翻译文件中没有这个 Key，或者 Key 写错了

解决：
1. 检查 en_us.json 中是否有这个 Key
2. 检查你的翻译文件中的 Key 是否与 en_us.json 完全一致（包括大小写）
3. JSON 格式是否错误（多一个逗号都会导致整个文件不生效）
```

### 7.2 JSON 格式验证失败

```
原因：多了一个逗号、少了一个引号、用了单引号

验证方法：
1. 复制 JSON 内容到 https://jsonlint.com/
2. 点击 Validate 检查格式是否正确
3. 按提示修复错误
```

> 💡 推荐使用 VS Code 编辑器，安装 JSON 格式化插件，会自动提示格式错误。

### 7.3 中文显示为乱码

```
原因：文件编码不是 UTF-8

解决：
1. 用 VS Code 打开文件
2. 右下角点击编码（如 GBK）
3. 选择 "Reopen with Encoding" → "UTF-8"
4. 保存
```

### 7.4 翻译文本太长被截断

```
原因：Minecraft UI 空间有限

解决：
1. 尽量缩短翻译
2. 中文通常比英文短，利用好这个优势
3. 长文本考虑放到 Tooltip 中
```

### 7.5 `%s` 顺序不对导致显示错误

```json5
// en_us.json 中的原文
{
  "message.modid.kill": "%1$s killed %2$s with %3$s"
  // 显示：Steve killed Zombie with Sword
}
```

```json5
// zh_cn.json（正确）
{
  "message.modid.kill": "%1$s 用 %3$s 杀死了 %2$s"
  // 显示：Steve 用 Sword 杀死了 Zombie
}
```

```json5
// zh_cn.json（错误 ❌ 改变了顺序，含义不清）
{
  "message.modid.kill": "%1$s 杀死了 %2$s 用 %3$s"
  // 中文语序别扭，但还是可读
}
```

### 7.6 不同语言的字符长度差异

| 语言 | 特点 | 注意 |
|------|------|------|
| 英文 | 单词较长 | GUI 可能溢出 |
| 中文 | 简短精炼 | 比较省空间 |
| 日语 | 混合汉字假名 | 长度适中 |
| 韩语 | 韩文字节较长 | 可能溢出 |
| 德语 | 单词极长 | 最容易溢出 |

> 💡 如果 UI 空间紧张，设计时尽量留出足够的边距。
