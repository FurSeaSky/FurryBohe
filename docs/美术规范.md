## 二、目录结构

### 2.1 资源文件存放位置

```
src/main/resources/
├── assets/
│   └── 你的modid/
│       ├── textures/          ← 贴图文件（.png）
│       │   ├── block/         ← 方块贴图
│       │   ├── item/          ← 物品贴图
│       │   ├── entity/        ← 实体贴图
│       │   ├── gui/           ← GUI 贴图
│       │   └── environment/   ← 环境/天空盒贴图
│       ├── models/            ← 模型文件（.json）
│       │   ├── block/         ← 方块模型
│       │   └── item/          ← 物品模型
│       ├── lang/              ← 语言文件（翻译人员负责）
│       │   ├── en_us.json
│       │   └── zh_cn.json
│       └── sounds/            ← 音效文件（.ogg）
└── META-INF/
    └── mods.toml              ← 模组元数据（不要动）
```

### 2.2 命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 方块贴图 | `block_名称.png` | `block_iron_ore.png` |
| 物品贴图 | `item_名称.png` | `item_steel_ingot.png` |
| 实体贴图 | `entity_名称.png` | `entity_zombie_boss.png` |
| GUI 贴图 | `gui_名称.png` | `gui_background.png` |
| 方块模型 | `block_名称.json` | `block_iron_ore.json` |
| 物品模型 | `item_名称.json` | `item_steel_ingot.json` |

> ⚠️ **全部使用小写字母 + 下划线**，不要用大写字母或空格


## 三、图片规范

### 3.1 贴图通用要求

| 项目 | 要求 |
|------|------|
| 格式 | `.png`（必须是 PNG 格式）|
| 尺寸 | 16×16、32×32、64×64、128×128、256×256、512×512 |
| 颜色模式 | **RGB**（不要用 CMYK）|
| 透明通道 | 支持 RGBA（透明背景用 PNG-24）|
| 分辨率 | **72 DPI** |
| 文件大小 | 单个不超过 **1MB** |

### 3.2 方块贴图要求

|------|------|
| 尺寸 | 16×16（标准）或 32×32（高清）|

| 数量 | 6 面（上下左右前后）或复用 |

### 3.3 物品贴图要求

| 位置 | 居中显示，四周留白 1-2 像素 |
| 风格 | 符合原版物品风格（像素画）|

### 3.4 实体贴图要求

| 项目 | 要求 |
|------|------|
| 尺寸 | 64×64 或 128×128 或 256×256 |
| 格式 | 展开 UV 贴图（实体模型 UV）|
| 风格 | 符合 Minecraft 原版实体风格 |

### 3.5 贴图层级要求

| 内容 | 图层命名 | 说明 |
|------|----------|------|
| 主纹理 | `texture` | 基础颜色层 |
| 阴影层 | `shadow` | 叠加阴影 |
| 高光层 | `highlight` | 叠加高光 |
| 法线贴图 | `_n` 后缀 | 如 `block_iron_n.png`（如有）|

> 如果是 PSD 源文件，图层按上述命名，导出时只导出合并后的 PNG。


## 四、模型规范

### 4.1 方块模型（JSON）

**标准格式：**

```json
{
  "parent": "minecraft:block/cube_all",
  "textures": {
    "all": "modid:block/block_iron_ore"
  }
}
```

**自定义模型：**

```json
{
  "parent": "minecraft:block/cube",
  "textures": {
    "up": "modid:block/block_machine_top",
    "down": "modid:block/block_machine_bottom",
    "north": "modid:block/block_machine_front",
    "east": "modid:block/block_machine_side",
    "south": "modid:block/block_machine_side",
    "west": "modid:block/block_machine_side"
  }
}
```

### 4.2 物品模型（JSON）

**标准格式：**

```json
{
  "parent": "minecraft:item/generated",
  "textures": {
    "layer0": "modid:item/item_steel_ingot"
  }
}
```

**手持/3D 物品：**

```json
{
  "parent": "minecraft:item/handheld",
  "textures": {
    "layer0": "modid:item/item_steel_sword"
  }
}
```

### 4.3 模型文件命名要求

```
✅ 正确：
block_iron_ore.json
item_steel_ingot.json

❌ 错误：
blockIronOre.json    （用了大写）
Block_Iron_Ore.json  （用了大写）
钢铁矿石.json         （用了中文）
```

### 4.4 模型路径与贴图对应关系

```
模型文件路径：
assets/modid/models/block/block_iron_ore.json

贴图路径：
assets/modid/textures/block/block_iron_ore.png

引用格式：
"modid:block/block_iron_ore"
         ↑          ↑
      贴图路径    文件名（无扩展名）
```


## 五、提交流程

### 5.1 提交前检查清单

提交前，请逐项确认：

- [ ] 贴图是 `.png` 格式
- [ ] 贴图尺寸是 16 的倍数（16、32、64、128、256）
- [ ] 物品贴图背景是透明的
- [ ] 文件名全部**小写字母 + 下划线**
- [ ] 文件放在正确的目录下（block/item/entity 分类正确）
- [ ] JSON 模型中的贴图路径引用了正确的文件
- [ ] 贴图风格符合 Minecraft 原版风格
- [ ] 没有提交 PSD/AI 等源文件（只提交导出的 PNG）

### 5.2 提交信息格式

```bash
feat(assets): 添加[具体内容]

示例：
feat(assets): 添加钢铁矿石方块贴图和模型
style(assets): 更新武器贴图风格
feat(assets): 添加新怪物实体贴图
```

### 5.3 PR 标题格式

```bash
[feat] 添加钢铁系列物品贴图和模型
[style] 更新 UI 按钮贴图
```

### 5.4 PR 描述模板

```markdown
## 变更类型
- [x] feat（新资源）
- [ ] style（样式调整）
- [ ] fix（修复）

## 变更说明
（描述添加/修改了什么贴图或模型）

## 资源清单
| 文件 | 说明 |
|------|------|
| block_iron_ore.png | 钢铁矿石方块贴图 |
| item_steel_ingot.png | 钢铁锭物品贴图 |
| block_iron_ore.json | 钢铁矿石方块模型 |

## 预览截图
（附上游戏内截图）

## 相关 Issue
（关联的任务编号，如有）
```


## 六、常见问题

### 6.1 贴图在游戏中显示紫色/黑色方块

```
原因：贴图路径写错了，或贴图文件名与模型 JSON 中引用的名称不一致

检查：
1. 贴图文件名是否全部小写 + 下划线
2. 模型 JSON 中 textures 路径是否正确
3. 贴图是否放在正确的目录下
4. 贴图格式是否为 PNG
```

### 6.2 物品贴图背景不是透明的

```
原因：导出时没有保留透明通道

解决：导出为 PNG-24（保留透明通道）
不要导出为 JPG（不支持透明）
```

### 6.3 贴图模糊/不清晰

```
原因：使用 RGB 模式时颜色管理或尺寸缩放问题

解决：
1. 确保原图分辨率是 16 的倍数
2. 不缩放贴图尺寸
3. 保存时使用 PNG 无损格式
4. 禁用颜色管理配置文件（在 Minecraft 中无效）
```

### 6.4 文件名有大写字母导致游戏崩溃

```
原因：Minecraft Forge 1.20.1 的资源路径区分大小写

解决：全部改为小写 + 下划线

错误：IronOre.png → 正确：iron_ore.png
```

### 6.5 模型显示为透明/不可见

```
检查 JSON 中 parent 是否写对：
- 方块模型：通常用 block/cube_all
- 物品模型：通常用 item/generated（手持物品用 item/handheld）

检查 textures 里的 key 是否正确：
- cube_all 用 "all"
- cube 用 "up", "down", "north", "east", "south", "west"
- generated 用 "layer0"
```

### 6.6 不能直接覆盖原版贴图

```
❌ 不要覆盖原版贴图路径

✅ 自己模组独立命名：
assets/你的modid/textures/block/xxx.png

❌ 不要放到：
assets/minecraft/textures/block/xxx.png
```

> ⚠️ 如需修改原版贴图，请使用资源包（Resource Pack）方式，不要直接在模组中覆盖。
